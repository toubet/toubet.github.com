1. 安装Git Bash

  • 官网下载地址

2. 安装Node.js，并将npm修改为国内镜像源

安装Node.js

  • 官网下载地址

  • 勾选同意后，即可点击next，后续修改安装位置，一直next即可。

  • 任一位置单击鼠标右键，点击“Git Bash Here”，打开git bash，输入“node -v”后回车。出现node版本信息即安装成功。

npm修改为国内镜像源

使用淘宝cnpm安装，在打开的git bash中输入如下命令回车

npm install -g cnpm --registry=https://registry.npm.taobao.org

安装软件时使用：

cnpm install 软件名

3.安装Hexo，并初始化博客

Hexo安装

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。Hexo官网有相关介绍和安装教程，也可以直接往下看。

在git bash输入如下命令回车

cnpm install -g hexo-cli

出现如下界面则安装成功

输入 hexo -v 查看Hexo版本

Blog初始化

选一个位置新建一个文件夹，为避免不必要的麻烦，文件夹名字尽量用英文，路径也不要有中文。虚拟机只弄了C盘，可以放在其他盘。如下所示

打开文件夹，在文件夹里面右键，点击“Git Bash Here”。打开“Git Bash”

然后输入如下命令初始化Blog。

hexo初始化命令：

hexo init

初始化成功后会出现以下文件或文件夹

node_modules：依赖包
public：存放的是生成的页面
scaffolds：命令生成文章等的模板
source：用命令创建的各种文章
themes：主题
_config.yml：整个博客的配置
db.json：source解析所得到的
package.json：项目所需模块项目的配置信息

初始化成功后依次输入如下命令：

hexo g
hexo s

然后打开浏览器，在地址栏输入下面的地址

http://localhost:4000/

即可本地访问hexo的默认博客界面

在“Git Bash”按“Ctrl+c”则关闭。至此hexo初始化成功。
常用命令：

命令 效果
hexo new “blog name” 生成博客文档，为markdown格式
hexo generate 生成静态文件。
hexo server 启动服务器。默认情况下，访问网址为： http://localhost:4000/。
hexo deploy 部署网站到服务器。
hexo clean 清除缓存文件 (db.json) 和已生成的静态文件 (public)。在某些情况如果发现对站点的更改无论如何也不生效，可能需要运行该命令。

更多命令，参考Hexo官网

markdown相关可以参考MarkDown语法

在博客文件夹内右键打开“Git Bash”，输入如下命令回车

hexo new “我的第一篇博客”

命令执行完成后会在博客目录下的 \source\_posts目录下是生成一个同名的markdown文件。

打开这个文件就可以写博客了

tags是本文所属标签，categories是类别，推荐使用VScode编辑器编写markdown文档。保存文件后，依次运行如下命令可以查看效果：

hexo g
hexo s

二. 将博客部署到GitHub

1. 注册

如果没有账号可以去GitHub官方网站进行注册

为免麻烦，用户名用英文。github已放弃IE浏览器，支持的浏览器有：Chrome， Firefox， Safari和 Microsoft Edge。

按提示进行人机验证，如果长时间无法加载出来，可以试试其他浏览器，如果都不行，可以试试手机端。

接下来的信息填写可以拉到后面点击跳过。

2. 新建仓库

登陆后新建一个公开的仓库，仓库名为”用户名.github.io”

3. 配置Git Bash

任意位置打开Git Bash，依次输入如下两行命令

git config --global user.name "输入你注册的用户名"
git config --global user.email "输入你注册的邮箱"

将命令依次输入到Git Bash中回车执行

4. 设置ssh公钥

在Git Bash输入下面的命令，引号内是你注册的邮箱。然后按3次回车即可。

ssh-keygen -t rsa -C "xxxxx@xxxxx.com"

查看公钥

cat ~/.ssh/id_rsa.pub

将公钥复制，到Github平台，打开个人设置，ssh公钥部分粘贴并确定。

title部分可填可不填。

5. 上传到Github

打开创建的仓库

点击ssh，点击后面的复制按钮

打开博客配置文件”_config.yml”。找到deploy部分，改为如下图所示，将复制的ssh链接粘贴到repo：之后， 注意所有冒号后面都需要空一格。

填写好之后保存退出

在博客目录下右键打开Git Bash，输入如下命令安装deploy

cnpm install hexo-deployer-git --save

然后输入如下命令将博客部署到Github仓库

hexo d

部署完成即可在Github仓库中看到你的博客文件

可以在setting一栏看到自己的访问链接，如果链接后面比案例多了“/”的话，那是因为你的仓库名不是“用户名.github.io”，可以在仓库的管理里面修改。

启动后即可访问自己的博客

写完博客文档后，hexo g生成静态文件，hexo s本地预览，预览没问题后Hexo d上传到Github仓库
